1 Task decomposition Unstructured input gets split into distinct tasks, each spawned as a separate parallel agent.
2 Spawns separate agents via skill Uses the "spawn-agent-pane" skill to launch independent agents in their own terminal panes — easier to track and debug than sub-agents. Link to skill in post.
3 Context-rich delegation From a vague request, the HQ agent infers file paths, project conventions, and scope from its config — then passes all of that to each agent.
4 Less downtime Multiple agents running means you rotate between them as each one needs input, instead of waiting on one.
